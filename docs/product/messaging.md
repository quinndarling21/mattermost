# Messaging

Messaging centers on posts, threads, mentions, and delivery behavior.

## Posts and threads

Threads can auto-follow when enabled. Collapsed reply threads can be disabled, defaulted off, defaulted on, or always on. This repository defaults collapsed threads to always on in server config.

Users can draft messages, schedule posts when the licensed feature is available, and use message priority when it is enabled by admins.

## Mentions and urgent delivery

Mattermost supports user, channel, `@here`, and group mentions. Large channel mentions can require confirmation before sending. Message priority can add persistent or repeating notifications with configured recipient, interval, and count limits.

## Join and leave messages

Admins can control whether the server posts join and leave messages. Users can override some related behavior in Account Settings.

## Self-deleting messages

The web app includes self-deleting message UI. Treat this as a managed feature area and verify license and admin settings before documenting it as available to all users.
