# Users and permissions

Permissions control what users, guests, admins, and delegated admins can do.

## Built-in roles

The server defines built-in roles for system admins, system managers, user managers, custom group admins, shared channel managers, read-only admins, team roles, channel roles, and guest roles.

## Permission scopes

Permissions apply at different scopes:

- System
- Team
- Channel
- Group
- Playbook
- Run

## Permission schemes

System Scheme defines the default permissions. Team Override Schemes let admins make exceptions for specific teams.

## Delegated granular administration

Custom system roles can grant a subset of administrative permissions. Use them when a user needs to manage a resource area without becoming a full system admin.

## Channel moderation

Channel moderation can separately control posting, reactions, mentions, bookmarks, and member management for members and guests.

## Admin guidance

Use roles for broad capability sets, permission schemes for team-level exceptions, and channel moderation for channel-specific restrictions.
