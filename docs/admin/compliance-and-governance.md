# Compliance and data governance

Compliance settings cover retention, export, auditability, privacy, and controlled collaboration.

## Retention

Admins can configure global and custom data retention policies. Retention policies affect how long data remains available and should be reviewed before changing storage, backup, or legal hold processes.

## Export and audit

Message export and audit logging are managed through System Console. Export settings affect how administrators retrieve historical content for compliance and review workflows.

## Governance controls

Related admin areas include:

- Custom terms of service.
- Content flagging and data spillage workflows.
- Classification markings.
- Secure connections and shared channels.
- IP filtering where licensed.

## Admin guidance

Document governance changes with the affected audience. Compliance officers need behavior, retention, export, and audit implications. End users usually need only visible workflow changes.
