# Lexicon

How to name Mattermost things correctly, which words to use, and which to avoid. Used together with [voice-and-tone.md](voice-and-tone.md) during every content review.

## Product and feature names

Use these exact forms.

- **Mattermost** - the product. One word, capital M. Never "MatterMost," "Matter Most," or lowercase in prose.
- **Channels** - the messaging surface. Capitalize when referring to the product feature; lowercase "a channel" when referring to a single channel.
- **Playbooks** - checklist-driven workflows (e.g., incident response). Capitalize the feature; "a playbook" / "a playbook run" for instances.
- **Teams** - a workspace grouping of channels and members.
- **Boards** - project/task boards, where applicable to the deployment.
- **Integrations** and the **plugin framework** - how Mattermost connects to other tools. A single add-on is "a plugin" or "an integration."
- **Self-hosted** - hyphenated as an adjective ("self-hosted deployment"). The act is "self-hosting."
- **Mattermost Cloud** - the managed offering. Don't write "the cloud" to mean Mattermost.
- **System Console** - the admin settings UI. Two words, capitalized.
- **System Admin**, **Team Admin**, **member**, **guest** - roles. Capitalize admin role names as shown.
- **Editions:** refer to the community/open-source edition and commercial editions accurately. Don't imply a feature is free if it requires a license.

## Capability words we mean precisely

- **Secure / security** - back it with the mechanism (self-hosting, access control, data residency, deployment model). Never as a bare adjective.
- **Open source** - the licensed, publicly developed project. Two words as a noun; hyphenate as an adjective ("open-source platform").
- **Data sovereignty / data control** - the customer owns and controls their data and where it runs.
- **Air-gapped** - fully network-isolated deployment. Only use when accurate.
- **Compliance** - name the specific framework or requirement; never claim certifications we don't hold.

## Use / avoid

Prefer the left, avoid the right.

- Use **"helps teams"**; avoid **"empowers teams"** (overused).
- Use **"brings together" / "in one place"**; avoid **"seamlessly integrates."**
- Use **"fast" / "real-time"**; avoid **"lightning-fast," "blazing-fast."**
- Use **"improves" / "speeds up"**; avoid **"supercharges," "turbocharges," "unlocks."**
- Use **"reliable" / "well-tested"**; avoid **"robust," "powerful," "world-class."**
- Use **"use" / "connect"**; avoid **"leverage," "utilize."**
- Use **"flexible" / "configurable"**; avoid **"frictionless," "effortless."**
- Use **"a new approach"** (only if true); avoid **"revolutionary," "game-changing," "next-generation."**
- Use **"self-hosted"**; avoid **"on-prem" in customer-facing copy** (fine internally; "self-hosted" reads clearer).
- Use **"because [reason]"**; avoid bare **"simply" / "just"** (they hand-wave over real work).

## Forbidden patterns

Reject copy that does any of these.

1. **Hype stacking** - two or more superlatives in a sentence ("the world's most powerful, revolutionary platform").
2. **Fear-based security marketing** - implying disaster to drive a sale ("Don't be the next breach," "hackers are coming for your data").
3. **Absolute security claims** - "unhackable," "100% secure," "breach-proof," "completely safe." Security is never absolute.
4. **Unsourced numbers** - any statistic, percentage, "X% faster," "trusted by N teams," or comparison without a citation.
5. **Undefined jargon** - buzzwords with no concrete meaning ("synergy," "paradigm shift," "holistic," "frictionless").
6. **False scarcity / pressure** - "act now," "limited time," "don't miss out."
7. **Lock-in framing** - language that contradicts self-hosting and data control.
8. **Misnaming the product** - any spelling other than "Mattermost," or capitalizing generic words as if they were features.

## Spelling and style defaults

- US English.
- Oxford comma.
- Spell out acronyms on first use, then abbreviate.
- "log in" (verb), "login" (noun/adjective). Same pattern for "set up" / "setup."
